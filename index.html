Wie funktioniert dieses Panel?
1. Datenhaltung (Firebase): Im normalen Live-Betrieb nutzt dieses System eine Google Firebase NoSQL-Datenbank. Darüber werden das Team-Management, die Zugriffsrechte und die ToDo-Listen sicher und in Echtzeit synchronisiert.
2. Server-Interaktion (REST-API): Das Panel interagiert direkt mit unserem FiveM-Spieleserver (republica.com). Klicken wir auf eine Aktion (z.B. "Geld setzen" oder "Inventar leeren"), sendet dieses Frontend einen HTTP-Request an die Server-API. Das Lua-Skript auf dem Server fängt den Request ab und wendet ihn sofort live im Spiel auf den ausgewählten Spieler an.
Für diesen Showcase wurden statische Test-Daten (Spieler, Inventare, Todos) geladen. API-Requests werden simuliert, um die Funktionalität auch offline fehlerfrei zu demonstrieren.
